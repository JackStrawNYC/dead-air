# Overlay Pool Audit Report

**Date:** 2026-03-05
**Total overlays:** 365 base + 52 parametric = **417 registered entries**
**Target:** Cull to ~250 high-quality, distinctive overlays

## Methodology

Each overlay scored on:
1. **Thematic fit** — Does it belong in a Dead concert visualization?
2. **Visual distinctiveness** — Is it meaningfully different from similar overlays?
3. **Energy coverage** — Does the pool have balanced low/mid/high representation?
4. **Production quality** — Is it a polished component or a filler entry?

## Executive Summary

The registry has **~120 overlays that should be culled**, primarily:
- **Thematic misfits:** Objects with no connection to music, counterculture, or concert visuals (Anvil, Sextant, Hydraulic, Piston)
- **Redundant entries:** Multiple overlays doing the same thing (7+ time-keeping devices, 8+ retro navigation instruments, 5+ aquatic creatures)
- **Low-distinctiveness filler:** Generic objects that add visual noise without meaning (ConveyorBelt, Transistor, Altimeter)

## Implementation Notes

To cull overlays:
1. Remove entries from `OVERLAY_REGISTRY` in `overlay-registry.ts`
2. Remove from `ENABLED_OVERLAYS` set (if present)
3. Keep the component files — they can be re-enabled later
4. Re-run `npx tsx scripts/generate-overlay-schedule.ts` to regenerate schedules without culled overlays
5. No component files need to be deleted — the registry gates selection

The culling is **non-destructive**: component code stays, only registry entries are removed. Any overlay can be restored by re-adding its registry entry.
